# Hueprint

[Permalink: Hueprint](https://github.com/iamsatz/HuePrint#hueprint)

**Design token kits for AI-builders.** Browse curated brand kits, preview on real UI components, download AI-ready files. Paste into Claude, v0, Cursor, or Bolt — get branded output instantly.

* * *

## What It Does

[Permalink: What It Does](https://github.com/iamsatz/HuePrint#what-it-does)

1. **Browse** — curated design token kits organized by industry
2. **Preview** — see tokens applied to real UI components (buttons, cards, forms, nav)
3. **Export**— download 3 files per kit:

   - `system-prompt.md` — AI-readable design brief (paste into Claude/v0/Cursor)
   - `tokens.css` — shadcn-compatible CSS variables
   - `tokens.json` — universal DTCG format

## Why It Exists

[Permalink: Why It Exists](https://github.com/iamsatz/HuePrint#why-it-exists)

AI tools generate functional UI but apply generic colors. Hueprint kits include **usage rules**, not just color values — so AI tools know _how_ to apply your brand, not just _what_ the colors are.

* * *

## Tech Stack

[Permalink: Tech Stack](https://github.com/iamsatz/HuePrint#tech-stack)

- **React 19** \+ TypeScript
- **Vite** (build)
- **Tailwind CSS v4** (styling)
- **shadcn/ui** (component primitives)
- **Lucide React** (icons)
- **chroma.js** (color utilities)
- **Vercel** (deployment)

## Getting Started

[Permalink: Getting Started](https://github.com/iamsatz/HuePrint#getting-started)

### Prerequisites

[Permalink: Prerequisites](https://github.com/iamsatz/HuePrint#prerequisites)

- Node.js 20+
- npm or pnpm

### Install

[Permalink: Install](https://github.com/iamsatz/HuePrint#install)

```
git clone https://github.com/your-username/hueprint.git
cd hueprint
npm install
```

### Dev Server

[Permalink: Dev Server](https://github.com/iamsatz/HuePrint#dev-server)

```
npm run dev
```

Open [http://localhost:5173](http://localhost:5173/)

### Build

[Permalink: Build](https://github.com/iamsatz/HuePrint#build)

```
npm run build
npm run preview
```

### Deploy

[Permalink: Deploy](https://github.com/iamsatz/HuePrint#deploy)

Push to `main` — Vercel auto-deploys.

## Project Structure

[Permalink: Project Structure](https://github.com/iamsatz/HuePrint#project-structure)

```
hueprint/
├── public/kits/          # Kit data (JSON per kit)
├── src/
│   ├── components/
│   │   ├── landing/      # Hero, HowItWorks, Footer
│   │   ├── kit-browser/  # KitGrid, KitCard, IndustryFilter
│   │   ├── kit-detail/   # KitDetail, ColorPalette, ComponentPreview
│   │   ├── preview/      # PreviewButton, PreviewCard, PreviewInput, etc.
│   │   ├── export/       # ExportModal, FormatSelector, CopyButton
│   │   └── ui/           # shadcn primitives
│   ├── lib/              # Utils, contrast checking, export format generators
│   ├── types/            # TypeScript interfaces
│   └── styles/           # Global CSS
├── PRD.md                # Product requirements
├── LICENSE               # MIT
└── README.md             # This file
```

## Kit Data Format

[Permalink: Kit Data Format](https://github.com/iamsatz/HuePrint#kit-data-format)

Each kit in `public/kits/` follows this structure:

```
{
  "id": "saas-modern",
  "name": "Modern SaaS",
  "industry": "SaaS",
  "description": "Professional, tech-forward, trustworthy.",
  "colors": {
    "light": { "primary": "#01696f", "..." : "..." },
    "dark": { "primary": "#2dd4bf", "..." : "..." }
  },
  "typography": { "..." : "..." },
  "spacing": { "..." : "..." },
  "borderRadius": { "..." : "..." },
  "shadow": { "..." : "..." }
}
```
